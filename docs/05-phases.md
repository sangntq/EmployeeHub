## Phase 0 — プロジェクトセットアップ

**目標**: チーム全員がローカルで開発・テストを実行できる状態にする。

### タスク

#### リポジトリ・構成
- [ ] モノレポ構成の決定（`frontend/` / `backend/` / `docs/`）
- [ ] `.gitignore`, `.editorconfig`, `README.md` 作成

#### バックエンド（FastAPI）
- [ ] Python仮想環境・`pyproject.toml`（Poetry）設定
- [ ] FastAPIプロジェクト初期構成（`app/`, `routers/`, `models/`, `schemas/`, `services/`）
- [ ] Alembicマイグレーション設定
- [ ] `docker-compose.yml`（PostgreSQL + FastAPI + Redis）
- [ ] 環境変数管理（`.env.example`）

#### フロントエンド（React）
- [ ] Vite + React + TypeScript プロジェクト初期化
- [ ] ディレクトリ構成（`pages/`, `components/`, `hooks/`, `api/`, `stores/`）
- [ ] ESLint / Prettier 設定
- [ ] 状態管理ライブラリ選定（Zustand 推奨）
- [ ] UIコンポーネントライブラリ選定（Ant Design / shadcn/ui 推奨）

#### CI/CD
- [ ] GitHub Actions: Lint + Test（push時）
- [ ] Dockerイメージビルド確認

### 完了基準
- `docker-compose up` で全サービスが起動する
- `/docs`（Swagger UI）にアクセスできる
- フロントエンドの空ページが `localhost:3000` で表示される

## Phase 5 — AI自然言語検索

**目標**: 顧客からのテキストをそのままAIに解析させ、検索条件を自動生成できる。

### バックエンド
- [ ] LLM API連携モジュール（`services/ai_parser.py`）
  - プロンプト設計（スキルマスタをコンテキストに渡す）
  - 構造化JSON出力の保証（Pydanticバリデーション）
  - タイムアウト・エラーハンドリング
- [ ] AI解析API（`POST /search/ai-parse`）
- [ ] AI検索ログ保存（`is_ai_search=true`）

### フロントエンド
- [ ] AI自然言語検索画面（S-04）
  - テキスト入力エリア
  - AI解析中のローディング表示
  - 抽出結果の確認・編集UI（タグ形式）
  - 「この条件で検索」ボタン

### 完了基準
- 日本語の自然文から条件が正しく抽出される
- スキルマスタに存在しない語句が `unmatched_terms` に表示される
- 抽出結果を手動で修正できる
