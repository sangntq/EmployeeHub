# 要件定義書

**プロジェクト名**: EmployeeHub
**作成日**: 2026年2月28日
**バージョン**: 1.1

---

## 1. プロジェクト概要

### 1.1 背景

日本向けITサービスを提供する会社において、ベトナム・日本に数千名のエンジニアを擁する。現状、各エンジニアのスキル・経歴・稼働状況を一元管理する仕組みがなく、新規案件発生時の人材マッチングや全社的な技術力の把握が困難である。

### 1.2 目的

| # | 目的 |
|---|------|
| 1 | 新規案件に最適な人材を迅速に検索・マッチングする |
| 2 | 全社の人材稼働状況をリアルタイムで可視化する |
| 3 | 会社・個人の技術力を定量的に評価・把握する |
| 4 | 資格・ビザの取得・更新情報を適切に管理する |

### 1.3 ロール定義

システムは以下の5ロールを持つ。ロールは階層的な承認権限を持つ。

| ロール | 説明 | 相当職位 |
|---|---|---|
| `member` | 一般メンバー（エンジニア） | 社員・スタッフ |
| `manager` | チームマネージャー | チームリーダー・PM |
| `department_head` | 部門長 | 部長・本部長 |
| `sales` | 営業担当 | 営業（人材検索・提案権限あり） |
| `director` | 役員・経営層 | 取締役・部門ディレクター |
| `admin` | システム管理者 | IT管理者 |

## 2. 機能要件

### 2.1 社員プロフィール管理

#### 基本情報
- 氏名（日本語・英語・ベトナム語）
- 社員番号・入社日・雇用形態（正社員 / 業務委託 / 契約社員）
- 所属拠点（ベトナム / 日本）・部署・チーム
- 連絡先（メール・電話・Slack ID）
- 顔写真

#### ビザ・在留資格管理
- ビザ種別（技術・人文知識・国際業務 / 特定技能 / 永住 等）
- ビザ有効期限
- 在留カード番号（日本勤務者）
- **期限アラート**: 有効期限60日前・30日前・7日前に自動通知

#### 雇用・契約情報
- 契約開始日・終了日
- 給与グレード（表示は権限制御）
- 勤務形態（常駐 / リモート / ハイブリッド）

---

### 2.2 スキル管理

#### スキルマスタ
- カテゴリ別スキル定義（言語・フレームワーク・DB・クラウド・ツール・ドメイン知識）
- `admin` / `department_head` によるスキルの追加・編集

#### 個人スキル登録と承認フロー

```
[member] スキル自己申告
    ↓
レベル入力（1〜5段階）+ 証拠ファイル添付（任意：資格証明書画像/PDF）
    ↓
[manager / department_head] 承認 or 差し戻し
    ↓
承認済みスキルとして確定・プロフィールに反映
```

- スキルレベル: 1（入門）/ 2（基礎）/ 3（実務）/ 4（上級）/ 5（エキスパート）
- 実務経験年数・最終使用時期を記録
- 自己評価と上長承認値を両方保持し、差異を可視化
- 承認待ちスキルは「審査中」として表示

#### スキルマップ
- 全社・部署・チーム単位のスキル分布ヒートマップ
- 不足スキルの可視化（案件要件と現有スキルのギャップ分析）

---

### 2.3 プロジェクト経歴管理

#### プロジェクト情報
- プロジェクト名・顧客名（日本企業）・業種
- 期間（開始〜終了）
- 担当役割（PL / PM / SE / PG / QA / インフラ 等）
- 使用技術スタック
- チーム規模・自身の担当範囲
- 成果・実績の記述（日本語・英語）

#### スキルシート（経歴書）出力
- 日本のIT業界標準フォーマットに準拠した単一テンプレート
- **個人単位**: Excel / PDF出力
- **複数候補者まとめ出力**: 営業・顧客提案用に複数名のスキルシートを1ファイルにまとめて出力
- 出力対象候補者は検索結果から選択してバッチ出力可能

---

### 2.4 資格・認定管理

#### 登録フロー

```
[member] 資格情報入力 + 証明書画像/PDFアップロード
    ↓
[manager / department_head] 内容確認・承認
    ↓
承認済み資格として確定
```

#### 登録情報
- 資格名・発行機関・取得日・有効期限
- 証明書ファイルのアップロード（PDF / 画像）
- 資格レベル・スコア（JLPT N1〜N5、AWS認定レベル等）

#### 主要資格カテゴリ（プリセット）
- **日本語**: JLPT（N1〜N5）、BJT
- **クラウド**: AWS / Azure / GCP 認定資格
- **プロジェクト管理**: PMP、IPA（基本・応用・高度）、ITIL
- **ネットワーク / セキュリティ**: CCNA、CompTIA
- カスタム資格の追加も可能

#### アラート
- 資格有効期限の自動通知（90日前 / 30日前）
- 本人・所属 `manager` への通知

---

### 2.5 稼働状況管理

#### ステータス定義

| ステータス | 説明 |
|---|---|
| アサイン中 | 現在プロジェクトに参画中 |
| フリー（即時） | すぐにアサイン可能 |
| フリー（予定） | 〇〇日以降にフリーになる予定 |
| 社内業務 | 社内プロジェクト・研修中 |
| 休職中 | 産休・病休等 |
| 退職予定 | 退職日が確定している |

#### アサイン管理
- 現在のアサイン案件・参画期間
- 複数案件への部分アサイン（例：A案件60% + B案件40%）
- フリー予定日の登録・変更

### 2.6 人材検索・マッチング

**利用可能ロール**: `sales` / `manager` / `department_head` / `director` / `admin`

#### 2.6.1 AIナチュラルランゲージ検索（自然言語検索）

営業担当者が顧客とのやり取り（メール・チャット・商談メモ等）をそのままテキストで貼り付けると、AIが要件を自動抽出して検索条件を生成する機能。

**ユースケース例:**

```
【顧客からのメール原文（貼り付け）】
来月からJavaのバックエンド開発者が2名必要です。
Spring Bootの経験が3年以上あって、AWSも使えると嬉しいです。
日本語はN2以上で、できればPMの経験もある方だと助かります。
フルリモートで参画できますか？
```

↓ AI処理

```json
{
  "extracted_criteria": {
    "skills": [
      { "name": "Java",        "level_min": 3, "required": true  },
      { "name": "Spring Boot", "experience_years_min": 3, "required": true },
      { "name": "AWS",         "required": false }
    ],
    "japanese_level": "N2以上",
    "roles": ["バックエンドエンジニア", "PM経験歓迎"],
    "work_style": "フルリモート",
    "headcount": 2,
    "availability": "翌月から"
  },
  "summary": "Java / Spring Boot 3年以上、AWS歓迎、日本語N2以上、フルリモート、2名、翌月から"
}
```

↓ ユーザーが確認・修正

↓ 通常の検索実行

**フロー詳細:**

```
① テキスト入力（日本語・英語・ベトナム語いずれも可）
② AIが要件を構造化データとして抽出
③ 抽出結果をUI上で確認・手動修正・追加
④ 「この条件で検索」で通常検索へ移行
⑤ 検索結果表示（マッチ度スコア付き）
```

**抽出対象の項目:**

| 抽出項目 | 例 |
|---|---|
| 必須スキル / 推奨スキル | Java必須、AWSあれば尚可 |
| 経験年数 | 3年以上 |
| 日本語レベル | N2以上 |
| 勤務形態 | フルリモート / 常駐 |
| 役割・ポジション | バックエンドエンジニア、PL経験者 |
| 稼働開始時期 | 翌月 / 即日 / ○月○日 |
| 必要人数 | 2名 |
| 業種・ドメイン | 金融、EC、製造 等 |
| 資格 | AWS認定、JLPT N2 等 |

**AI実装方針:**
- バックエンド（FastAPI）からLLM API（Claude / OpenAI）を呼び出し
- プロンプトエンジニアリングで構造化JSON出力を保証
- 抽出結果は確定前に必ずユーザーが確認・編集できるUIを提供（誤抽出対策）
- 検索ログに「AI検索」フラグを付与して追跡可能にする

#### 2.6.2 通常検索（フィルター検索）

#### 検索条件
- スキル（複数選択・AND / OR条件）・スキルレベル指定
- 経験年数・プロジェクト経験業種
- 稼働ステータス・フリー予定日
- 拠点（ベトナム / 日本）・勤務形態
- 日本語レベル（JLPT等）・ビザ種別
- 資格保有

#### 検索結果
- マッチ度スコア表示（必須スキル充足率）
- 一覧 / カード表示の切り替え
- 候補者を選択してスキルシートを一括ダウンロード（複数名まとめ出力）
- 検索条件の保存・再利用

### 2.7 ダッシュボード・レポート

#### 経営ダッシュボード（`director` / `admin`）
- 稼働率（全社 / 拠点 / 部署）
- フリー人材数・スキル別分布
- 直近1〜3ヶ月のフリー予定推移

#### 部門ダッシュボード（`manager` / `department_head`）
- 担当チーム・部門の稼働状況
- チームスキル分布・資格取得状況

#### スキル分析レポート
- 全社スキルレーダーチャート
- スキルカテゴリ別人数分布
- 資格取得状況・取得推移

#### アラートダッシュボード
- ビザ・資格の期限切れ一覧（期限順ソート）
- 承認待ちスキル・資格一覧
- 未更新プロフィール一覧（90日以上更新なし）

#### エクスポート
- Excel / CSV形式でのデータ出力（日本語対応）
- 月次・四半期レポートの自動生成

### 2.10 認証・ログイン

#### 対応認証方式
1. **Googleアカウント認証（OAuth 2.0）**: 会社のGoogleワークスペースアカウントでシングルサインオン
2. **社内発行アカウント**: メールアドレス + パスワード（`admin`が発行・管理）

#### セキュリティ要件
- JWTによるセッション管理（アクセストークン + リフレッシュトークン）
- パスワードはbcryptでハッシュ化
- ログイン失敗5回でアカウント一時ロック
- 操作ログ（ログイン・ログアウト・主要操作）を記録

## 3. 技術スタック

| レイヤー | 技術 | 備考 |
|---|---|---|
| フロントエンド | React + TypeScript + Vite | SPA構成 |
| バックエンド | Python + FastAPI | REST API、OpenAPI自動生成 |
| データベース | PostgreSQL | 自社管理、全データを一元管理 |
| 認証 | OAuth 2.0（Google）+ JWT | 自社アカウントも JWT で統一 |
| ファイルストレージ | S3互換（MinIO 等）| 証明書・顔写真の保存 |
| コンテナ | Docker + Docker Compose | 開発・本番環境の統一 |

### アーキテクチャ概要

```
[React SPA]  ←→  [FastAPI]  ←→  [PostgreSQL]
                     ↓
              [File Storage (S3互換)]
              [メールサービス (SMTP/SendGrid)]
```

- フロントエンド・バックエンドは完全分離（CORS設定で接続）
- バックエンドはRESTful JSON API（OpenAPI / Swagger UI自動生成）
- フロントエンドはVercel / Nginx 等の静的ホスティング
- バックエンドはDockerコンテナで運用

## 4. 非機能要件

### 4.1 パフォーマンス
- 人材検索レスポンス: 2秒以内（5,000名規模）
- ダッシュボード初期表示: 3秒以内

### 4.2 可用性
- 稼働率: 99.5%以上（月間計画停止を除く）
- データバックアップ: 日次

### 4.3 セキュリティ
- 通信の暗号化（HTTPS / TLS 1.2以上）
- 操作ログの記録・保存（1年間）
- 個人情報保護法（日本）・ベトナム個人情報保護規定への準拠

### 4.4 多言語・ローカライゼーション
- UI言語: 日本語 / ベトナム語（切替可能）
- データ入力: 日本語・英語・ベトナム語混在対応
- タイムゾーン: JST（日本）/ ICT（ベトナム）対応

### 4.5 スケーラビリティ
- 初期: 〜1,000名
- 想定成長: 5,000名まで追加コスト最小で対応可能な設計
