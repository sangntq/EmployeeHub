# BUG-2026-03-01-001: /employees 画面が何も表示されない

## 概要
`http://localhost:3000/employees` にアクセスすると画面が何も表示されない。

## 重大度
LOW（操作手順の問題）

## 症状
- ブラウザから `/employees` に直接アクセスしても画面が空白に見える

## 期待動作
- 社員一覧テーブルが表示される

## 再現手順
1. ブラウザで新しいタブを開く（ログインしていない状態）
2. `http://localhost:3000/employees` に直接アクセス

## 調査結果

### 根本原因
ユーザーが未ログイン状態で `/employees` にアクセスした場合、`ProtectedRoute` が `/login` にリダイレクトする。
ログインフォームが表示されているが、ユーザーがそれを「何も表示されない」と誤認した。

### 確認事項
- Backend: 正常稼働（`http://localhost:8000` → 200 OK）
- Frontend: 正常稼働（Vite dev server ポート3000起動済み）
- DB: 1名の社員データあり（`システム管理者`）
- API 認証後のレスポンス: 正常（200 + 1件）

### 関連コードバグ（対応見送り）
`frontend/src/pages/Login/index.tsx` L22-30 にて `AuthUser` オブジェクトの
フィールドマッピングが型定義と不一致。
- `name`, `employeeId` が未設定（`undefined` になる）
- `nameJa`, `nameEn`, `employeeNumber` は `AuthUser` 型に存在しない
- 影響: AppHeader でユーザー名が空、プロフィールリンクが `/employees/undefined`
- ユーザー判断: 今回は対応見送り

## 解決方法
ログイン後に `/employees` にアクセスする。

1. `http://localhost:3000/login` にアクセス
2. `admin@employeehub.local / Admin1234!` でログイン
3. `/employees` に移動

## テスト結果
- 対応不要（ユーザー操作の問題）
