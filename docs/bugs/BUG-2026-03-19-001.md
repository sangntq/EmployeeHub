# BUG-2026-03-19-001: 資格管理概要ページの検索が常に空を返す

## 概要
`/certifications` ページの検索ボックスにキーワードを入力しても、資格名での検索結果が一件も返ってこない。

## 重大度
**MEDIUM** — 機能は使えるが検索が無効状態。回避策なし。

## 症状
- 資格名（例: "AWS", "JLPT"）で検索 → 0件
- 社員名で検索 → 正常に返る
- キーワードをクリアすると全件表示に戻る

## 期待動作
資格名・社員名のどちらで検索しても、該当する資格カードが表示される。

## 根本原因
`certification_service.get_company_cert_overview()` のサーチ処理（[backend/app/services/certification_service.py:228-236](../../backend/app/services/certification_service.py)）で、`or_()` 条件に `CertificationMaster.name` が含まれていなかった。

また、`category` フィルターが未指定の場合は `CertificationMaster` テーブルへの JOIN が行われないため、その条件を追加しても SQL エラーになっていた。

```python
# 修正前: CertificationMaster.name が未検索
or_(
    Employee.name_ja.ilike(like_pattern),
    Employee.name_en.ilike(like_pattern),
    EmployeeCertification.custom_name.ilike(like_pattern),
    # ← CertificationMaster.name が抜けていた
)
```

## 修正内容
| # | ファイル | 変更内容 |
|---|---------|---------|
| 1 | `backend/app/services/certification_service.py` | `search` が指定され `category` が未指定の場合に `CertificationMaster` を left outer join し、`or_()` に `CertificationMaster.name.ilike()` を追加 |

```python
# 修正後
if search:
    like_pattern = f"%{search}%"
    if not category:  # category 未指定時は JOIN が不足しているため追加
        query = query.join(
            CertificationMaster,
            EmployeeCertification.certification_master_id == CertificationMaster.id,
            isouter=True,
        )
    query = query.where(
        or_(
            Employee.name_ja.ilike(like_pattern),
            Employee.name_en.ilike(like_pattern),
            EmployeeCertification.custom_name.ilike(like_pattern),
            CertificationMaster.name.ilike(like_pattern),  # ← 追加
        )
    )
```

## テスト結果
- `pytest -k "cert"` → **12 passed** ✅
- リグレッションなし

## 備考
- 社員名（`name_ja` / `name_en`）による検索は正常に動作していたため、影響は「資格名で検索できない」のみ
- `category` + `search` の組み合わせは、category ブロックで既に JOIN 済みのため影響なし
